# 部署手册

这份文档用于把项目从“本地开发运行”整理成“可以用 Docker 启动和演示”的形态。

## 1. 部署目标

```text
别人拉取项目
-> 配置 .env
-> docker compose up --build
-> 访问 http://127.0.0.1:8001/web
-> 能使用客服工作台、RAG、LangGraph 实验区
```

## 2. 本地 Docker 启动

先确认项目根目录有 `.env`：

```powershell
Copy-Item .env.example .env
```

默认不开启真实 LLM：

```env
LLM_ENABLED=false
```

启动：

```powershell
docker compose up --build
```

后台启动：

```powershell
docker compose up -d --build
```

访问：

```text
网页工作台：http://127.0.0.1:8001/web
接口文档：http://127.0.0.1:8001/docs
健康检查：http://127.0.0.1:8001/health
```

健康检查正常返回：

```json
{"status":"ok"}
```

停止：

```powershell
docker compose down
```

## 3. 文件职责

```text
Dockerfile：定义镜像怎么构建，安装 Python 依赖，并启动 FastAPI。
docker-compose.yml：定义容器端口、环境变量、数据目录挂载。
.dockerignore：排除 .env、虚拟环境、数据库、日志等不该进镜像的文件。
```

## 4. 数据持久化

Docker 容器里使用：

```text
APP_DB_PATH=/app/data/complaints.db
```

`docker-compose.yml` 会把本机目录挂载到容器：

```text
./data:/app/data
./logs:/app/logs
```

这样设计的原因：

```text
容器删除后，数据库文件仍然保留在本机 data 目录。
镜像里不包含真实业务数据，避免把本地数据库上传到镜像或 GitHub。
```

## 5. LLM 配置

如果要在 Docker 中启用 DeepSeek：

```env
LLM_ENABLED=true
LLM_PROVIDER=deepseek
LLM_MODEL=deepseek-v4-flash
LLM_TIMEOUT_SECONDS=60
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=你的真实key
```

注意：

```text
.env 不会被 Git 提交。
.env 也不会被复制进 Docker 镜像。
docker compose 会在运行容器时读取本机 .env。
```

## 6. 生产启动命令

Dockerfile 中使用的是生产式启动命令：

```text
python -m uvicorn app.main:app --host 0.0.0.0 --port 8001 --app-dir .
```

它和开发命令的区别：

```text
开发命令带 --reload，适合本地改代码自动重启。
生产命令不带 --reload，运行更稳定。
```

## 7. 部署前检查

建议部署前运行：

```powershell
.\.venv311\Scripts\python.exe -m pytest tests\test_langgraph_agent.py tests\test_langchain_agent.py tests\test_langchain_rag_agent.py tests\test_langchain_tools.py tests\test_langchain_retriever.py tests\test_api_routes.py -q
node --check web\app.js
.\.venv311\Scripts\python.exe -m compileall app
```

还要确认没有提交敏感文件：

```powershell
git status --short
```

不要提交：

```text
.env
data/*.db
logs/*.log
practice/
```

## 8. 常见问题

### docker compose 命令不存在

说明 Docker Desktop 可能没有安装或没有启动。

先确认：

```powershell
docker --version
docker compose version
```

### 容器启动后访问不到网页

检查端口：

```text
http://127.0.0.1:8001/web
```

如果 8001 被占用，可以修改 `docker-compose.yml`：

```yaml
ports:
  - "8002:8001"
```

然后访问：

```text
http://127.0.0.1:8002/web
```

### 修改代码后 Docker 里没变化

重新构建：

```powershell
docker compose up --build
```

### 数据库想重置

停止容器：

```powershell
docker compose down
```

删除本地数据库：

```powershell
Remove-Item .\data\complaints.db
```

再启动：

```powershell
docker compose up --build
```

启动时 `init_db()` 会重新建表和初始化教学数据。
