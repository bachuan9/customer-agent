# 运行与测试手册

## 1. 本地启动

在项目根目录执行：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --app-dir . --port 8001
```

启动成功后访问：

```text
网页工作台：http://127.0.0.1:8001/web
接口文档：http://127.0.0.1:8001/docs
健康检查：http://127.0.0.1:8001/health
```

健康检查正常时返回：

```json
{"status":"ok"}
```

## 2. 环境变量

复制配置模板：

```powershell
Copy-Item .env.example .env
```

默认可以不开启 LLM：

```env
LLM_ENABLED=false
```

如需测试 DeepSeek Function Calling：

```env
LLM_ENABLED=true
DEEPSEEK_API_KEY=你的真实 key
```

注意：不要把 `.env` 或真实 API Key 提交到 GitHub。

## 3. 自动测试

运行核心测试：

```powershell
.\.venv\Scripts\python.exe -m pytest tests\test_agent_evaluation.py tests\test_agent_core.py tests\test_knowledge_rag.py tests\test_api_routes.py -q
```

当前期望结果：

```text
111 passed, 3 warnings
```

检查前端 JavaScript：

```powershell
node --check web\app.js
```

检查 Python 文件是否能编译：

```powershell
.\.venv\Scripts\python.exe -m compileall app
```

## 6. 常见问题

### 页面还是旧功能

原因通常是后端没有重启，或浏览器缓存了旧的 `web/app.js`。

处理方式：

```text
重启后端
刷新浏览器
必要时强制刷新 Ctrl + F5
```

### 接口返回 404

常见原因是后端还在跑旧代码。重启后端即可。

### LLM did not return a tool call

意思是大模型没有按 Function Calling 格式返回工具调用。系统会自动降级到规则 Agent，所以这不是项目崩溃。

### LLM request timed out

意思是大模型接口超时。系统会记录降级原因，并退回模板回复。

### Agent Evaluation 为什么默认关闭 LLM

因为自动评测需要稳定可重复，而真实 LLM 可能受网络、超时、模型返回格式影响。Agent Evaluation 主要测试确定性的意图识别、工具调用、RAG、多轮上下文和组合工具链路。
